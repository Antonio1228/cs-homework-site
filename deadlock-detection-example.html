Deadlock Detection Example (Step by Step)

Deadlock detection is used to determine whether a system is currently in a deadlocked state. Unlike Banker's algorithm (avoidance), detection finds deadlocks after they happen.

Given

We have 5 processes (P0 to P4) and 3 resource types (A, B, C).

Allocation Matrix

Process A B C
P0 0 1 0
P1 2 0 0
P2 3 0 3
P3 2 1 1
P4 0 0 2

Request Matrix (Current Request)

Request[i] shows the current resources that Pi is requesting (still needs right now to continue).

Process A B C
P0 0 0 0
P1 2 0 2
P2 0 0 0
P3 1 0 0
P4 0 0 2

Available Vector

        Available = (0, 0, 0)

Deadlock Detection Algorithm

We use Work and Finish: 

        Work = Available
        Finish[i] = false for all i (or true if Allocation[i] = 0)
        Repeat:
          Find i such that Finish[i] = false and Request[i] <= Work
          If found:
            Work = Work + Allocation[i]
            Finish[i] = true
        If no such i exists:
          All processes with Finish[i] = false are deadlocked

Step by Step Detection

Step 0: Initialize

        Work = (0, 0, 0)
        Finish = [false, false, false, false, false]

Step 1: Find a process with Request <= Work

Check each process request: 

        P0 Request = (0,0,0) <= Work(0,0,0) -> Yes
        
        Work = Work + Allocation(P0)
        Work = (0,0,0) + (0,1,0) = (0,1,0)
        Finish[P0] = true

Step 2

        Now Work = (0,1,0)
        
        P2 Request = (0,0,0) <= Work(0,1,0) -> Yes
        Work = Work + Allocation(P2) = (0,1,0) + (3,0,3) = (3,1,3)
        Finish[P2] = true

Step 3

        Now Work = (3,1,3)
        
        P3 Request = (1,0,0) <= Work(3,1,3) -> Yes
        Work = Work + Allocation(P3) = (3,1,3) + (2,1,1) = (5,2,4)
        Finish[P3] = true

Step 4

        Now Work = (5,2,4)
        
        P1 Request = (2,0,2) <= Work(5,2,4) -> Yes
        Work = Work + Allocation(P1) = (5,2,4) + (2,0,0) = (7,2,4)
        Finish[P1] = true

Step 5

        Now Work = (7,2,4)
        
        P4 Request = (0,0,2) <= Work(7,2,4) -> Yes
        Work = Work + Allocation(P4) = (7,2,4) + (0,0,2) = (7,2,6)
        Finish[P4] = true

Result

        Finish = [true, true, true, true, true]
        No deadlock detected.
        All processes can finish.

Conclusion

The deadlock detection algorithm tries to find processes whose current Request can be satisfied by Work (available resources). If every process can eventually be marked Finish = true, there is no deadlock. Otherwise, remaining unfinished processes are deadlocked.
