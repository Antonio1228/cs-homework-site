First and Follow Set Example (Step by Step)

FIRST and FOLLOW sets are used in compiler design (parsing) to build predictive parsing tables for LL(1) grammars.

Grammar

            E  → T E'
            E' → + T E' | ε
            T  → F T'
            T' → * F T' | ε
            F  → ( E ) | id

Rules Used

FIRST Rules

  • FIRST(terminal) = terminal
  • If X → ε, then ε ∈ FIRST(X)
  • If X → Y1 Y2 ... Yk, add FIRST(Y1) (except ε). If ε ∈ FIRST(Y1), continue to Y2, etc.

FOLLOW Rules

  • Put $ in FOLLOW(start symbol)
  • If A → α B β, then add FIRST(β) (except ε) to FOLLOW(B)
  • If A → α B (or A → α B β and ε ∈ FIRST(β)), then add FOLLOW(A) to FOLLOW(B)

Step 1: Compute FIRST Sets

FIRST(F)

            F → ( E ) | id
            FIRST(F) = { (, id }

FIRST(T')

            T' → * F T' | ε
            FIRST(T') = { *, ε }

FIRST(T)

            T → F T'
            FIRST(T) = FIRST(F) = { (, id }

FIRST(E')

            E' → + T E' | ε
            FIRST(E') = { +, ε }

FIRST(E)

            E → T E'
            FIRST(E) = FIRST(T) = { (, id }

Final FIRST Table

Non-terminal FIRST
E { (, id }
E' { +, ε }
T { (, id }
T' { *, ε }
F { (, id }

Step 2: Compute FOLLOW Sets

Initialize

            Start symbol: E
            FOLLOW(E) includes { $ }

Use productions to add symbols

From F → ( E )

            After E comes ')'
            So ')' ∈ FOLLOW(E)
            FOLLOW(E) = { $, ) }

From E → T E'

            After T comes E'
            Add FIRST(E') except ε => '+'
            So '+' ∈ FOLLOW(T)
            
            Since ε ∈ FIRST(E'), also add FOLLOW(E) to FOLLOW(T)
            FOLLOW(T) includes { +, $, ) }

From E' → + T E'

            After T comes E'
            Add FIRST(E') except ε => '+'
            So '+' ∈ FOLLOW(T) (already)
            
            Since ε ∈ FIRST(E'), also add FOLLOW(E') to FOLLOW(T)
            (We will finalize FOLLOW(E') soon and propagate)

From T → F T'

            After F comes T'
            Add FIRST(T') except ε => '*'
            So '*' ∈ FOLLOW(F)
            
            Since ε ∈ FIRST(T'), also add FOLLOW(T) to FOLLOW(F)
            FOLLOW(F) includes { *, +, $, ) }

From T' → * F T'

            After F comes T'
            Add FIRST(T') except ε => '*'
            So '*' ∈ FOLLOW(F) (already)
            
            Since ε ∈ FIRST(T'), also add FOLLOW(T') to FOLLOW(F)
            (We will finalize FOLLOW(T') soon and propagate)

FOLLOW(E')

            E → T E'
            E' is at the end, so FOLLOW(E) ⊆ FOLLOW(E')
            FOLLOW(E') = { $, ) }

FOLLOW(T')

            T → F T'
            T' is at the end, so FOLLOW(T) ⊆ FOLLOW(T')
            FOLLOW(T') = { +, $, ) }

Final FOLLOW Table

Non-terminal FOLLOW
E { $, ) }
E' { $, ) }
T { +, $, ) }
T' { +, $, ) }
F { *, +, $, ) }

Conclusion

FIRST sets determine which terminals can start strings derived from a non-terminal. FOLLOW sets determine which terminals can appear immediately to the right of a non-terminal in sentential forms. These sets are essential for building LL(1) parsing tables.
