LRU Page Replacement Example (Step by Step)

LRU (Least Recently Used) page replacement algorithm replaces the page that has not been used for the longest time when a new page needs to be loaded into memory.

How LRU Decides the Victim Page

When a page fault happens and the frames are full, LRU chooses the page that was used least recently (the one with the oldest last-used time) and replaces it with the new page.

Conclusion

LRU often performs better than FIFO because it uses recent usage history. However, it may be more expensive to implement because the system must track page usage order.
