Belady's Anomaly Example (FIFO) with Step by Step Solution

Introduction

Belady's anomaly is the counterintuitive situation in some page replacement algorithms (notably FIFO) where increasing the number of page frames can increase the number of page faults for the same reference string. Below is a simple worked example that demonstrates this behavior using the FIFO page replacement algorithm.

Given

                                Reference string: 1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5
                                Consider two cases using FIFO:
                                  - Case A: 3 frames
                                  - Case B: 4 frames

Comparison and Conclusion

- Page faults with 3 frames (FIFO): 9
- Page faults with 4 frames (FIFO): 10

Increasing the number of frames from 3 to 4 caused the number of page faults to increase (9 -> 10). This illustrates Belady's anomaly: under FIFO page replacement, adding more frames can sometimes increase the number of page faults rather than decrease them.
